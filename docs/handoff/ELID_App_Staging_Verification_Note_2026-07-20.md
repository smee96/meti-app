## B. 개선 제안 3건 (여유 있을 때 반영해주시면 됩니다)

### B-1. card 메시지에 card_name JOIN (낮음)
- 현재 `GET /chat/:roomId/messages`의 card 타입 메시지에 `card_id`만 있고 명함 이름이 없습니다.
- 앱은 '명함'이라는 범용 레이블로 폴백 표시 중이라 동작엔 문제없지만, **`card_name`(cards.name JOIN)을 내려주시면** "이규한 님의 명함"처럼 표시할 수 있습니다.

### B-2. card 타입일 때 방 목록 last_message 채우기 (낮음)
- 명함 공유가 마지막 메시지면 `GET /chat`의 `last_message`가 null로 내려와 앱이 "메시지가 없습니다"로 표시합니다.
- 발송 푸시 body처럼 **"명함을 보냈습니다."** 등 대체 문구를 `last_message`에 저장해주시면 목록 UX가 자연스러워집니다. (이미지/파일은 파일명이 들어와서 괜찮습니다)

### B-3. 날짜 포맷 통일 — ISO 8601 UTC 권장 (낮음, 장기)
- 현재 응답에 두 형식이 혼재합니다: `"2026-07-18 10:36:47"`(D1 기본, 타임존 표기 없음) / `"2026-07-19T00:00:00.000Z"`(ISO).
- 앱은 "타임존 없으면 UTC로 간주" 파서로 방어 처리했지만, 웹·향후 클라이언트도 같은 함정을 밟을 수 있어 **신규/수정 API부터 ISO 8601 UTC(`...Z`)로 통일**을 권장합니다. 기존 필드 변경은 불필요합니다(앱은 양쪽 다 처리).

## C. 현황 공유

- 핸드오프(채팅·푸시·§5 결제/NFC) 앱 작업은 **FCM 클라이언트를 제외하고 전부 완료·검증**됐습니다: 폴링(방 5초/목록 10초), 신고/차단 UI, 이미지 업로드(첨부는 제품 결정으로 이미지만 — PDF/Word 파일 첨부는 제외), 명함 공유 메시지, 메시지 삭제, 보관기간 안내, 403 교환 유도, NFC 신청/내역 화면, 포인트 외부 브라우저 충전+OTT.
- **FCM**: Firebase 프로젝트 소유 계정은 대표 확인 대기 중입니다. 확정되면 앱팀이 생성 후 v1 서비스 계정 키를 전달하겠습니다 — 서버는 시크릿 등록만 하면 되는 상태 그대로 유지해주세요.
