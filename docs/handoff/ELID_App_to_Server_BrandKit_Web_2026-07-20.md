## 2. 웹 대시보드 이슈 2건 (앱 → 웹 포인트 충전 진입 시)

재현 경로: 앱 포인트 화면 → "포인트 충전" → 외부 브라우저(에뮬레이터 Pixel 5 / Chrome)로
`{webBaseUrl}/app/points?ott={token}` 오픈. 앱은 `POST /auth/web-session-token`으로 발급받은 원타임 토큰을 `?ott=`로 전달합니다(회신 2026-07-16 §C-2 합의사항).

### 2-1. 자동 로그인(ott 교환) 실패 정황
- 대시보드가 렌더되지만 **모든 사용자 데이터가 비어 있음**: `안녕하세요, -님`, 내 명함 `-`, 소속 그룹 `-`, 포인트 `-`, 현재 플랜 `-`.
- 비로그인 폴백이라면 로그인 화면으로 보내야 하는데, 빈 대시보드가 그대로 노출됩니다.
- 확인 요청: ① `?ott=` → 세션 교환이 staging에서 동작하는지 ② 교환 실패/미로그인 시 빈 대시보드 대신 로그인 리다이렉트 처리.
- 참고로 진입 URL은 `/app/points`인데 표시된 화면은 대시보드였습니다 — 라우팅/리다이렉트도 함께 봐주세요.

### 2-2. 모바일 반응형 깨짐
- 모바일 뷰포트(360~412px)에서 "대시보드" 제목과 카드들이 **왼쪽으로 잘려** 나갑니다(가로 오버플로). 사이드바/컨테이너가 모바일에서 접히지 않는 것으로 보입니다.

## 3. (신규) `template_id` 디자인 접미사 안내 — 웹 공유 페이지 파싱 1줄 수정 요청

앱에 명함 **레이아웃 디자인 선택**이 추가됐습니다. 저장 형식:
- 기존과 동일: `default`, `dark` 등 (클래식 레이아웃 — **하위호환, 변경 없음**)
- 신규: `{팔레트}__{디자인}` 예) `default__center`, `ocean_coral__leftbar` (디자인: `center` | `leftbar`)

서버는 자유 문자열이라 그대로 저장되며(스키마 변경 불필요), **웹 공유 페이지만** 팔레트 매칭 전에 `template_id.split('__')[0]`으로 팔레트를 취해주세요. 미수정 시 신규 디자인 명함이 웹에서 default 팔레트로 폴백됩니다(깨지진 않음). 레이아웃 자체는 웹 반영 불필요 — 팔레트만 유지되면 됩니다.
